#   5.  Products  → Add 1 Hazardous product
#   6.  Products  → Add 1 Fragile product
#   7.  Products  → List all products
#   8.  Products  → View product detail (Standard)

#  10.  Stock     → Receive goods (Standard)
#  11.  Stock     → Transfer Standard to STORAGE
#  12.  Stock     → Transfer Perishable to COLD_STORAGE

#  14.  Stock     → Transfer Fragile to STORAGE
#  15.  Stock     → Dispatch goods (Standard)
#  16.  Stock     → Post adjustment (Standard)
#  17.  Stock     → Movement history (Standard)
#  18.  Reports   → Inventory summary
#  19.  Reports   → Stock value by zone
#  20.  Reports   → Expiry report
#  21.  Reports   → Movement history report (date range)
#  22.  Settings  → Add user
#  23.  Settings  → Change PIN
#  24.  Settings  → Delete user
#  25.  Main      → Exit
#
# NOTE ON ZONE SELECTION (transferZones):
#   getZoneSelection() prints a numbered list of ALL zones and reads
#   an integer index. Zones are inserted in this order:
#     1) Inbound Dock A      (RECEIVING)
#     2) Main Storage Hall   (STORAGE)
#     3) Outbound Dock B     (DISPATCH)
#     4) Refrigerated Unit 1 (COLD_STORAGE)
#     5) Chemical Store 1    (HAZMAT)
#   So entering 2 selects Main Storage Hall, 4 = Refrigerated, etc.
#
# NOTE ON DELETE:
#   deleteProduct and deleteZone require MongoDB-generated IDs known
#   only at runtime. Test those manually after a first run.
